git 基本操作

2020-07-27

git 常用操作

不会的可以查看git –help

创建工作空间

clone

把远程代码克隆到本地

  • 克隆默认分支
  • 克隆指定分支

init

初始化git目录，使用场景主要是创建本地git管理，如果需要上传到远程服务器，则需要指定远程仓库地址

处理当前修改的文件

add

把快照文件添加到缓存

  • 添加单个文件
  • 添加全部文件

mv（少用）

重命名或者移动文件

rm（少用）

删除文件

restore（少用）

撤销修改，同git checkout -- 命令，我一般用的是checkout --

commit

把暂存的文件提交版本区

reset

还原文件到某个节点

  • 取消暂存
  • 把文件重设到某个节点，即把head重制到某个节点

cherry-pick

这个是很好用的一个命令，作用是把复制别的commit节点并应用到当前分支，使用场景比如a分支需要使用b分支的某个节点，这时候直接在a分支上去cherry-pick b分支的节点j即可

检查历史和状态

bisect（少用）

使用二分法查找出现错误的commit节点，多用于找bug。

diff

与HEAD节点对比文件

  • 与工作空间对比
  • 与缓存空间进行对比

log

查看提交日志

show（少用）

查看提交的详细内容

status

查看工作空间的文件状态

分支管理

branch

查看、创建、删除分支

switch 切换分支（少用）

checkout

切换、创建分支的集合，还有就是处理未暂存文件

  • 切换分支，同git switch
  • 撤销修改

merge/rebase

这两个的作用都是合并分支，合并分支，会经常出现冲突，主要不同是两个命令生成的节点树不一样。rebase类似于执行一系列的cherry-pick命令，最后生成的节点树是一条线，待合并分支的节点都在主分支节点后。 而merge生成的是两条线，而且节点还是保留在原有位置，如果有冲突的话会自动生成一个merge节点。

  • rebase
  • merge

tag

新建、删除标签

与远程仓库交互

fetch

pull

在当前分支上执行fetch 和 merge

push

把本地内容推到服务器

git流程

正常迭代

  • 基于master拉取代码
  • 本地修改完成后推送到远程仓库
  • 符合上线标准后，有条件可以别人帮忙review代码并提交合并请求
  • 合并到master
  • 新建tag

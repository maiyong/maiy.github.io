Kotlin调用Java

Kotlin在设计时就考虑了java互操作性。两者可以很自然的互相调用。

Getter和Setter

java中以名称以get或者set开头的方法，在Kotlin中会表示为属性。如果是Boolean访问器的是以is和set开头的方法，表示为isXX。Kotlin不支持只有一个Setter。

返回void方法

java中void是不返回任何值，Kotlin调用了void方法时会返回Unit。

将Kotlin中是关键字的java标识符进行转义

Kotlin中的关键字(is、object、in等)在java中用到了可以适用进行转义。

空安全与平台类型

Java中所有引用都有可能为null，这使得Kotlin对来自java的对象要求严格空安全是不现实的，同时也容易造成空指针崩溃。因此我们可以使用null检测。

Kotlin中的泛型

  • Java通配符转换
    Foo<? extends Bar> 转换成 Foo<out Bar!>!
    Foo<? super Bar> 转换成 Foo<in Bar!>!
  • Java原始类型转换成星投影
    List转换成List<*>!，即List<out Any?>!
    与Java一样，Kotlin的运行时也不保留泛型的，也就是说无法使用is来判断泛型。

java数组

Kotlin的数组是不型变的，也就是说不允许Array<String>赋值给一个Array<Any>。
Kotlin可以使用特化的类(IntArray、DoubleArray、CharArray等)与Java进行交互，它们与Array类无关，并且会编译成Java原生类型数组以获得最佳性能。

不允许使用中缀表示法调用Java方法

受检异常

Kotlin中不会强迫try catch，所以调用一个声明受检异常的方法时，Kotlin不会进行try catch。

对象方法

当Java类型导入到Kotlin中时，java.lang.object的所有引用都成了Any，但是因为Any只声明了toString()、hashCode()、equals()，如果要用到java.lang.object的其他成员时，就要用到扩展。

  • wait()/notify()

  • getClass()

  • clone()
    需要使用clone时，需要继承Cloneable

3. 实例字段

如果需要在Java中显示Kotlin的属性字段，就需要使用@JvmField注解对其标注。如果一个属性有幕后字段、非私有、没有open/override或者const修饰符并且不是被委托的属性，那就可以用@JvmField进行注解。

4. 静态字段

在命名对象或伴生对象中声明的Korlin属性会在该命名对象或包含伴生对象的类中具有静态幕后字断。这些字段通常是私有的，可以使用一下方式暴露出来。

  • @JvmField注解
    使用@JvmField标注这样的属性使其成为与属性本身具有相同可见性的静态字段。

  • lateinit修饰符
    在命名对象或伴生对象中的一个延迟初始化具有与属性setter相同可见行的静态幕后字断。

  • const修饰符
    用const标注的属性在Java中会成为静态字段，以下下几种情况在Java中都是静态字段。

5. 静态方法

Kotlin可以为命名对象或伴生对象注解@JvmStatic则会在Java处生成静态方法。

6. 可见性

Kotlin的可见性以下列方式映射到Java：

  • private成员编译成private成员。private的顶层声明编译成包级局部声明。
  • protected保持protected。
  • internal在Java处会变成public，internal类的成员会通过名字修饰，使其更难以在Java中意外使用到。

  • public还是保持public.
